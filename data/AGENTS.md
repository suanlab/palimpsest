# data

## Purpose
Local data storage for raw downloads, processed datasets, API caches, and Neo4j database files. This directory is gitignored — data is acquired via scripts.

## Key Files

| File | Description |
|------|-------------|
| `author_nodes.parquet` | Author node dataset (~1.6 GB) |
| `author_nodes_gzip.parquet` | Gzip-compressed author nodes (~813 MB) |
| `field_nodes.parquet` | Field/subfield node dataset |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `raw/` | Raw data from external sources (OpenAlex, arXiv, DBLP, Crossref, AMiner, Retraction Watch) |
| `processed/` | Preprocessed parquet/JSON — figures, graph data, Neo4j import CSVs, retraction analysis |
| `cache/` | API response cache (diskcache backend) |
| `neo4j/` | Neo4j database files (managed by Docker) |

## For AI Agents

### Working In This Directory
- **Never commit data files** — this directory is gitignored
- Data is acquired via `scripts/collect_*.py` and `scripts/download_*.sh`
- Processed data is generated by `scripts/etl_*.py` and `scripts/analyze_*.py`
- Parquet format for large datasets; CSV for Neo4j imports
- Paths are configured via `research.utils.config.Settings.data_dir`

### Data Pipeline
```
raw/ (API downloads) → processed/ (ETL output) → neo4j/ (graph DB import)
                                  ↓
                          processed/figures/ (publication plots)
```

## Dependencies

### Internal
- `scripts/` — All data acquisition and ETL scripts
- `src/research/data/` — API clients that cache responses here
- `src/research/utils/config.py` — Path configuration
